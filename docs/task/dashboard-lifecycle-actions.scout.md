## A. Publish / teardown internals

### Reusable primitive
- `cleanupOldVersions(pageId, keepCount = 10)` ‚Äî `src/lib/staticExport/versionCleanup.ts:16`.
  Reads `PublishedPageVersion.metadata.blobs[].blobKey` (falls back to legacy `version.blobKey`),
  `del()`s each, deletes the version row. **`keepCount: 0` = delete ALL versions + blobs** ‚Äî the
  closest thing to a take-down primitive. Best-effort per-version try/catch; never throws.
- `src/lib/staticExport/renderPublishedExport.ts:547-556` ‚Äî on DB failure loops `allBlobs` and
  `del(b.blobKey)` for EVERY page blob. The correct multi-blob cleanup pattern.
- `src/app/api/publish/route.ts:520-528` ‚Äî outer rollback only `del(uploadedBlobKey)` (ROOT blob
  only). Weak; do NOT model unpublish on this one.

### Publish write order (invert this)
1. `publishedPage.update|create` (route.ts:209/242) ‚Äî content/theme/template snapshot
2. `project.upsert` status `'published'` (:278)
3. `publishState: 'publishing'` (:334)
4. `renderPublishedExport()` (:370) ‚Üí HTML + `uploadStaticSite` per page ‚Üí creates
   `PublishedPageVersion` (status `'active'`, `metadata.blobs=[{path,blobKey,blobUrl,sizeBytes}]`)
   ‚Üí `publishedPage.update` sets `publishState:'published'`, `currentVersionId`, `lastPublishAt`
5. KV: `atomicPublishWithRetry(pageId, domains, version, blobUrl, {extraRoutes})` (:416);
   domains = `publishSubdomainHosts(slug)` (lessgo.site + legacy lessgo.ai) + `canonicalDomain` if live
6. Redirects/slug-for (:434-455); fire-and-forget `cleanupOldVersions(pageId,10)` (:500) + blog sync (:507)

**Nothing is transactional.** No Prisma `$transaction` in the path. KV is `kv.pipeline()` ‚Äî
atomic-ish per call only. The spec's "atomic teardown" must be BUILT, not inherited.

### publishState machine
No machine module ‚Äî ad-hoc string writes on `PublishedPage.publishState`. Writers: `'publishing'`
(route.ts:336), `'failed'` (:480,:536), `'published'` (renderPublishedExport.ts:540).
**No path ever sets `'draft'`** (schema default only). `PublishedPage.isPublished` exists (limit
count, route.ts:229) and is **never written `false`**. Readers: `src/app/admin/page.tsx:337`,
`src/app/api/admin/kv/route.ts:63`.

### KV routes ‚Äî `src/lib/routing/kvRoutes.ts`
Key shapes: `route:{host}:{path}`, `redirect:{host}:/`, `slug-for:{host}`.
**Multiple keys per page**: one route key per (domain √ó path); domains = 2 subdomains + optional
custom domain; paths = `/` + every subpage/locale path.
- Write: `atomicPublish(...)` :121, `atomicPublishWithRetry(...)` :211, `setRoutes(entries)` :168,
  `writeRedirect(fromHost,toUrl,status=301)` :333, `writeSlugForHost(host,slug)` :369
- Delete: `removeRoutes(hosts: string[])` :354 ‚Äî **ROOT PATH ONLY** (`route:{host}:/`,
  `redirect:{host}:/`, `slug-for:{host}`). `deleteRoutes(keys: {host,path}[])` :184 ‚Äî arbitrary
  paths. `removeRedirect(fromHost)` :346, `removeSlugForHost(host)` :373
- ‚ö†Ô∏è **TRAP**: `removeRoutes` alone strands non-root routes (subpages/locales/blog). Precedent for
  the workaround: `src/app/api/domains/remove/route.ts:54-67` hand-patches `deleteRoutes` for
  `/blog`. Unpublish MUST enumerate subpage/locale paths ‚Äî available from
  `PublishedPageVersion.metadata.blobs[].path`. This is directly the "no orphaned KV routes" AC.

### Blobs
- `import { del } from '@vercel/blob'` (versionCleanup.ts:1, renderPublishedExport.ts:3,
  publishBlogPost.ts:11, dynamic at publish/route.ts:522)
- Key: `pages/{pageId}/{version}/{pageName}.html` (`blobUploader.ts:67`; root = `index`,
  subpages/locales use their path e.g. `nl/about`)
- **Every version has its own folder; every page/locale in a version its own blob.** Publish keeps
  up to 10 versions ‚Üí a full take-down may face ~10 versions √ó N pages of blobs.

### Existing take-down code
- `src/app/api/domains/remove/route.ts` ‚Äî closest structural model (authz ‚Üí Vercel `removeDomain`
  ‚Üí `removeRoutes`/`removeRedirect`/`deleteRoutes` ‚Üí DB clear ‚Üí best-effort blog re-sync). Removes
  only the domain; never touches blobs/versions/publishState.
- `src/lib/blog/publishBlogPost.ts` ‚Äî per-post unpublish; read for blob-then-route ordering convention.
- **No admin unpublish/delete endpoint, no script.** `cleanupOldVersions` is the only reusable
  primitive; everything else is net-new.

## B. Authz / routes / clone

### `assertProjectOwner` ‚Äî `src/lib/security.ts:57-110+`
```ts
assertProjectOwner(clerkId: string|null|undefined, tokenId: string,
  opts: { action: string; claimIfOrphan?: boolean; allowMissing?: boolean }): Promise<ProjectOwnerResult>
```
**Never throws** ‚Äî returns discriminated union (`security.ts:47-55`):
`{ok:true, isDemo, adminOverride, userRecord:{id}|null, project:{userId}|null}` | `{ok:false, status, error}`.
Ladder: demo token `lessgodemomockdata` ‚Üí allow; no clerkId ‚Üí 401; user missing ‚Üí 404; project
missing ‚Üí 404 unless `allowMissing`; owner ‚Üí allow; orphan (`userId==null`) ‚Üí allow (+claim if
`claimIfOrphan`); **admin non-owner ‚Üí allow + `logAdminOverride`**; else 403.

Canonical usage ‚Äî `src/app/api/projects/[tokenId]/route.ts:23-30`: `auth()` ‚Üí 401 if no clerkId ‚Üí
`assertProjectOwner(clerkId, tokenId, { action: 'projects.read' })` ‚Üí
`if (!access.ok) return NextResponse.json({error: access.error},{status: access.status})`.
Mutations pass `claimIfOrphan: true` (cf. saveDraft).

‚ö†Ô∏è **No opt-out flag for admin override.** Strict owner-only requires an explicit
`access.adminOverride === true ‚Üí 403` check. See ORCHESTRATOR DECISION D2.

### Project routes today

## ORCHESTRATOR DECISIONS (made unattended; log in plan, reviewer may challenge)

- **D1 ‚Äî custom-domain guard predicate = `publishedPage.customDomain !== null`**, regardless of
  `customDomainStatus`. Rationale: a domain in `pending_*`/`issuing_ssl`/`failed` still holds the
  `@unique` slot + a live Vercel registration; keying off `status === 'live'` would leak
  half-attached domains. Strictly safer reading of spec Q1=A.
- **D2 ‚Äî authz = `assertProjectOwner(..., { action, claimIfOrphan: true })`** on every mutation,
  exactly as spec Scope-IN mandates; explicitly NOT the `published-slug` hand-rolled widening.
  Its built-in admin override is **audited** (`logAdminOverride`) and is KEPT ‚Äî the spec's target
  was the *unaudited* bypass. Flagged for the founder's access-control human gate: "admin may
  delete/unpublish any project (logged)" is a policy call for the founder, not the pipeline.
  If the founder wants strict owner-only, add `access.adminOverride === true ‚Üí 403`.
- **D3 ‚Äî scope**: Domain settings + Archive menu items stay greyed (not in spec scope).
