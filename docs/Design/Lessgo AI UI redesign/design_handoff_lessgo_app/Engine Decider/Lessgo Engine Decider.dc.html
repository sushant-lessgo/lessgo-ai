Engine Decider — how one line becomes an engine

Before the work journey: which engine?

The current onboarding assumes one engine — work. In reality every incoming business is routed to one of five. These screens are the decider that runs inside Step 1: a one-liner in, a provisional engine out, and — only when we genuinely can’t tell — one plain question. The engine is a revisable belief, not a hard fork: inferred here, revised through ingestion, committed at the plan gate. When the engine resolves to work, today’s screens take over unchanged.

D4 When we genuinely don’t know — one plain question THE FIX

Fires for an unknown type or an ambiguous-known one — a branding studio can go work or trust (the cirkles case), and guessing wrong builds the wrong site, not the wrong color. So we hand the decision to the only real oracle: the human. We don’t ask “what kind of site” — we ask what wins the visitor over, which is the engine, in plain words. The likely answer is pre-selected; all five are shown, including the two we can’t build yet.

D5 When the honest answer is “not yet”

Picking place or quick-yes today isn’t a dead end and it’s never mis-served as something it isn’t. We say so plainly, offer a real human hand-off, and log the demand — every rejection is a signal, not a lost user. This is the opposite of a cold upfront waitlist: the gate is on understanding, after we already get you.

D6 Engine set — and still revisable to the plan gate

The decider ends by handing off to the engine’s own journey — for work, that’s today’s screens, unchanged.
