№ 01Identity

Type-as-art. Whitespace as confidence. Color as restraint.

Folio is for buyers who sell taste. They don't need to convince you with bullet points; they need to convince you that they understand how a thing should look. Where Meridian decorates with structure and Hearth decorates with breath, Folio decorates with nothing: rule lines, asterisks, a single italic word. The page is a printed magazine page that learned to scroll.

If you can remove a thing and the page still works, remove it. Then remove one more.

This is the system most likely to be misread by an AI generator as "boring." It is not boring. It is restrained. The difference is the type stack, the measure, the rhythm of rule lines, and the discipline to hold the accent at 3% of any viewport.

№ 02Typography

The single most important system in Folio. Three font-systems, picked by variant.

Variants

  • Serif — Source Serif 4 across display + body. Classical, literary, unhurried. The choice for hospitality, hospitality-adjacent, and long-form publishers.
  • Mixed — Source Serif display, Manrope body. The magazine standard. Default. Best for designers, photographers, taste-led DTC, premium consumer.
  • Grotesque — Manrope across display + body. Italics replaced with weight 300 contrast. The Swiss option — for studios that want rigor without warmth.

Semantic roles

  • eyebrow — 11px mono 500, tracking +0.22em, uppercase, ink-3. Bookended with rule lines or accent number.
  • folio — the printer's term for a page-top line: small mono caps with a date / volume / page no. Used at the top of any reading column.
  • lede — 22px serif italic, never sans even in Mixed variant. Max 32ch.
  • drop-cap — first paragraph of long-form sections. 5em serif (or 4.4em sans 600 in Grotesque). Always accent-deep.
  • callout — 28px serif italic, left rule in accent. Used once per section maximum.

Accent palettes (low chroma — restrained)

Nine palettes, all held at low chroma. Bone (the default) is true monochrome — accent is just a darker ink. The colored options nudge the system toward a sector without ever shouting. The system enforces "≤ 3% of viewport weight" through component rules — accent is allowed in: italic words, drop-caps, the brand's star mark, eyebrow numbers, callout rules, button hover. Forbidden in: card fills, large blocks, multiple icons.

№ 05Decoration

Five permitted motifs. No more.

  • Rule lines. 1px rule at 18% opacity. Above section headings, between rows, framing everything. The cards layer is replaced entirely by rule lines.
  • Folio numbers. № 03 instead of 03. Always with the numero glyph. Used for section indices and item indices.
  • Asterism. A single ✻ in accent-deep. Used as a brand mark and as the section break in long-form.
  • Italic accent words. One italic word per heading, in accent-deep. The italic does the work color usually does.
  • Drop-caps. One drop-cap per long-form section. 5em serif (4.4em weight-600 sans in Grotesque variant).

No icons. No emoji. No illustrations. If you need a picture, use a picture.

№ 06Components

Buttons, forms, cards, lists.

Buttons

  • Primary — ink fill, paper text, square corners (radius 0), 16px 24px, 14px sans 500, hover shifts to accent-deep.
  • Ghost — transparent, 1px rule border, ink text. Hover: border becomes ink.
  • Quiet — text only, ink-2 + bottom border on hover. For tertiary actions.

Cards (there are no cards)

Folio does not have cards. Where another system would card a feature, Folio rule-lines it. Where another system would card a pricing tier, Folio columns it with rule-line separators. The only "card-like" thing is the Index Row — and even that is a rule line above and below.

Forms

Inputs are bottom-bordered only. No fill, no surrounding box. Label is a small mono caps line above, placeholder text is italic serif. Focus pulls the bottom border to accent-deep.

Lists

Bullets are em-dashes (`—`), not dots, when set in serif body. In sans body, bullets are middle-dot, accent color, 22px size. Lists set in two different body fonts use the same em-dash treatment for consistency.

✻ from the journal

On the discipline of removing.

When we begin a project we make a list of every element a competitor has put on their page — every badge, every gradient, every floating widget. We then ask, of each, whether removing it would make the work weaker, or merely smaller. The answer, almost always, is the second.

Restraint is not the absence of decisions. It is the most decisions, made earlier, in quieter rooms. A page with one italic word in it has been argued over more than a page with twelve gradient cards. The italic word is the visible tip of a long debate; the gradients are the absence of one.

Tokens, copy & paste.

The full token surface for the renderer. The user-configurable axes are --accent / --accent-deep (palette, 9 options) and the family roles set per [data-variant] attribute (Serif · Mixed · Grotesque). Everything else is held.

:root tokens

/* Surface */
--paper:        oklch(0.985 0.004 85);   /* main */
--paper-1:      oklch(0.965 0.006 80);   /* secondary */
--paper-2:      oklch(0.94  0.008 75);
--ink:          oklch(0.16  0.008 60);   /* primary text */
--ink-2:        oklch(0.36  0.006 60);
--ink-3:        oklch(0.55  0.005 60);

/* Lines (the elevation system) */
--rule:         oklch(0.16 0.008 60 / 0.18);
--rule-soft:    oklch(0.16 0.008 60 / 0.10);
--rule-strong:  oklch(0.16 0.008 60 / 0.32);

/* Type */
--font-serif:   "Source Serif 4", Times, Georgia, serif;
--font-sans:    "Manrope", ui-sans-serif, system-ui, sans-serif;
--font-mono:    "JetBrains Mono", ui-monospace, monospace;

/* Radius — flat. */
--r-sm: 0;
--r-md: 0;
--r-lg: 2px;
--r-xl: 2px;

/* Section rhythm */
--sec-pad-y:   160px;       /* mobile: 96px */
--sec-pad-x:   clamp(28px, 5vw, 96px);
--max-w:       1320px;

variant overrides ([data-variant])

/* Serif — Source Serif everywhere */
html[data-variant="serif"]     { --font-display: var(--font-serif); --font-body: var(--font-serif); }

/* Mixed (default) — serif display, sans body */
:root                          { --font-display: var(--font-serif); --font-body: var(--font-sans); }

/* Grotesque — sans everywhere; italics → weight 300 */
html[data-variant="grotesque"] { --font-display: var(--font-sans);  --font-body: var(--font-sans); }
